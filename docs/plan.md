# Project Plan – Project 03

## Overview

This project is a two-page personal portfolio website built using HTML and CSS. The goal is to deliver a simple, professional website along with structured project documentation based on PMUPM principles.

---

## Risk Analysis (TAME Framework)

### T – Technical Risks
- Risk of broken links between pages.
- Risk of CSS not applying consistently across both pages.
- Risk of GitHub Pages not deploying correctly.

Mitigation:
- Test navigation manually before submission.
- Use one shared stylesheet for both pages.
- Verify GitHub Pages settings early.

---

### A – Audience Risks
- The site may feel too basic or too template-like.
- The purpose of the website may not be clear.

Mitigation:
- Write clear introductory content.
- Keep layout clean and professional.
- Ensure navigation is intuitive.

---

### M – Management Risks
- Running out of time close to deadline.
- Forgetting required documentation files.

Mitigation:
- Complete documentation before final submission.
- Use a checklist to verify requirements.
- Commit changes incrementally.

---

### E – Environmental Risks
- GitHub deployment delays.
- Internet issues during submission.

Mitigation:
- Submit early.
- Verify live site before turning in link.

---

## Work Breakdown Structure (Task List)

1. Define project topic and scope.
2. Create `index.html`.
3. Create `about.html`.
4. Create shared `style.css`.
5. Test navigation links between pages.
6. Create `scope.md`.
7. Create `plan.md`.
8. Create `retrospective.md`.
9. Update README with live link and documentation link.
10. Final testing and submission.

---

## Timeline

The project was completed in stages across multiple sessions:
- Website structure built first.
- Styling applied second.
- Documentation completed last.
- Final review and corrections before submission.

# Project Retrospective – Project 03

## What Went Well

The scope planning phase went very well. Defining the project clearly at the beginning made the build process more straightforward. The HTML structure was simple and clean, and using one shared CSS file ensured consistent styling across both pages.

The GitHub repository structure was organized properly, and the live deployment through GitHub Pages worked successfully.

---

## What Went Wrong

The main issue was incomplete documentation. While the website itself was functional, the planning documents were not fully written before submission. This created confusion during grading.

Time management toward the end of the project also became tighter than expected.

---

## What I Would Do Differently

Next time, I would:
- Complete all documentation before focusing on final polish.
- Use a checklist earlier in the process.
- Do a final repository review before submitting.

This project reinforced the importance of execution discipline and managing drift, as emphasized in PMUPM. Even when the technical product works, the documentation is equally important.
